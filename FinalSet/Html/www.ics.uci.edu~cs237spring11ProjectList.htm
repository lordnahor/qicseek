 Topic 5:  (Research Study) Agent-based Service Management for Wireless and Mobile Devices( 1 Student )  
     
  Offloading tasks from thin devices (wireless and mobile) onto other resource sufficient nodes is a promising and popular research topic now. Adapting to the current residual energy and moving patterns of a wireless and mobile device, a service scheduler can achieve power efficiency and better QoS performance. Imagine a situation that all service providers contact the same device at the same time with the same query frequently, this will put much overhead on the thin client with unnecessary cost. So, this project tries to provide an agent-based approach to managing a number of general services for each device. Since it keeps the current status of the mobile and wireless node, we name the agent as a "shadow" of the device.  Therefore, this project will include:
  • A survey about agent-based approach to providing services for wireless and mobile devices

  • Simulation of presented ideas
  • Report simulation results
  • A final report

 Topic 1:  On-line Profiling Based Energy Reduction Video Conferencing  
     

This project aims to use the information available to the encoder to reduce energy consumed at the decoders. In a typical video conference, one device encodes the video stream and provides the compressed data to many decoding devices. If the encoder can provide decoding time to each decoder, then the decoder can apply DVS (dynamic voltage scaling) very efficiently. However, the problem with this is that it is very difficult to get the exact decoding time since the decoding devices have different platforms such as IPAQ, Palm, Zaurus, Axim and so on. Therefore, in this project you will figure out which information is necessary for each decoding device to predict decoding time and how one can collect the information in the encoding device.

 Topic 1:  Querying in Sensor Networks (2 + 2 students)  
     
Sensors are typically deployed to gather data about the  physical world.  Since sensors are resource constrained, often sensor data is collected into a sensor database that resides at (more powerful) servers. A natural tradeoff exists between  the sensor resources (bandwidth, energy) consumed and the quality of data collected at the server. Blindly transmitting sensor updates at a fixed periodicity to the server results in a suboptimal solution due to the differences in stability of sensor values and due to the varying application needs that impose different quality requirements across sensors. Paper [HMV04] proposes adaptive data collection mechanisms for sensor environments that adjusts to these variations while at the same time optimizing the energy consumption of sensors.   
 
  • Project A. Quality-aware Aggregate Queries in Wireless Sensor Networks (2 students)
    • Paper [HMV04] only considers queries that request single sensor readings.  This project extends to aggregate queries which ask for the sum, average, min/max of a group of sensor readings. 
    • Hints: Explore query locality. subsequent aggregate queries may request data from a set of overlapping sensors, so obtaining an update from those sensors not only benefits current query, but also the next one.  In addition, sensors can coordinate with each other while deciding its state transitions.
    • Relevant papers: HMV04, OW00
       
  • Project B. Energy Efficient Data Collection in Wireless Sensor Networks (2 students)
    • There are basically three communication methods that sensor networks use: direct communication, multi-hop, or cluster-based. Paper [HMV04]  considers the scenario where direct communication between sensors and the server is more efficient than the other two manners. This project extends it to either multi-hop or cluster-based communication sensor networks (depending on your interests).  
    • Hints: Re-write sensor energy consumption formula in paper [HMV04] by taking into account energy consumption incurred due to the routing, and derive the optimal point where energy consumption is minimized.
    • Relevant papers:  HMV04, OW00, IGE00, HCB00
