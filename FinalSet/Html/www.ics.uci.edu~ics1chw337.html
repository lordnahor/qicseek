  1. What is a protocol?

    Well, when you're referring to computers (i.e., hacker jargon) protocol refers to any set of rules that allow different machines or pieces of software to coordinate with each other without ambiguity. But if you want to know the definition you would find in the Webster Dictionary, then the answer is an original draft, minute, or record of a document or transaction. A transaction, hmm...that sounds kind of like what the jargon definition is referring to--exchanging data via an agreement

    According to the Admin-Guide-to-Cracking-101 if you want to beat a hacker you must first learn the ways of a hacker. This document puts you through a typical scenario in which a person tries to gain illegal access to a system. Then it tells you methods of defeating such attempts.   4. What is fuzzy logic?
    Fuzzy logic is a method within Boolean logic whose purpose is to deal with the values that lie "between" completely true and completely false. Let me explain. In Boolean algebra, via a set of defined operators, a statement obtains a value of true or false. Depending on the values used for the variables in the statement a statement may be true all of the time, some of the time, or never. Fuzzy logic deals with the cases in which it is true some of the time.

  5. How do I write an HTML document?
    The letters HTML stand for HyperText Markup Language, so it makes sense that you create an HTML document via markups, which are statements enclosed in < and > (less than and greater than brackets). The purpose of HTML is to specify the structure of the document, not the appearance. Writing an HTML document is similar to programming in that you use words and symbols with predefined meanings to stand for something else.
