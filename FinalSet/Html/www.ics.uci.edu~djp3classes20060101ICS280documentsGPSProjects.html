  • Goals
    • The goal of this project is to give participants hands on experience with GPS error. I want participants to have a physical working sense of how reliable and accurate an off-the-shelf receiver is and compare that to reported characteristics. The result will be that participants will be better designers of location-aware systems.
  • Details

    • There are two GPS units available. They have been distributed to Wiwat and Trung.

  • Project 1: GPS Error survey
    • Collect position information from four locations:
      • Location 1: 6 hours of GPS traces from an open, unobstructed location. This will provide a baseline for what the "best you can expect" should be.
      • Location 2: 2 hours of GPS traces from an interesting obstructed location.
      • Location 3: 2 hours of GPS traces from an urban canyon.
      • Location 4: 15-30 minutes of GPS traces from a vehicle.
    • Analysis
      • Report on the parameters of the receiver. Single-dual frequency, channels, etc.
      • Visualize this data in several ways. 2-D Histograms, temporally, overlays on satellites, anything that helps to illuminate features of the collected signals.
      • Present an analysis of any interesting features in the visualization. Why does the analysis look the way it does? To what degree do the interference parameters that we discussed in class appear or not appear in the data. What is surprising about the data? What is expected about the data.
      • Given the results what applications can use the GPS receiver that you used? What applications can not use this receiver?
    • Deliverables
      • a 5-10 page paper describing the survey
      • a 15 minute presentation describing the survey
      • group self-evaluation
  • Project 2: Geocaching course
    • Develop a position-based treasure hunt that would be appropriate for prospective students to engage in
      • Select a number of locations on campus (~7)
      • Find the lat/long of those positions
      • Stash a weather-proof treasure or clue at each location. (In a plastic container probably)
      • The course should be designed in such a way that two teams can run the course at the same time competitively and there should be a disincentive for either team to steal or misplace the treasure once discovered.
    • Create a web-site that will support the treasure hunt. It can be interactive with information from the treasures.
    • Do one round of redesign: Reserve one or two members of your group as the "testers". Have that(those) person(s) run the course and then provide feedback to the group. Incorporate the feedback into the course.
    • Be creative
    • Deliverable
      • A geo-cache course that takes about an hour to run.
      • A website that can be used as a launching point for the course. You can assume that someone running the course has mobile-phone style access to the Internet. (simple-forms are okay)
      • Host a live run of the course for the other group.
      • a less than 10 page paper describing the layout of the course, and how the mechanisms of the course work. (Sufficient that the course can be rebuilt)
      • Group self-evaluation
  • Project 3: Offline DGPS software
    • This project can be done as a stand-alone project or as extra-credit
    • Write a piece of software that takes a basic GPS trace, applies off-line DGPS corrections from CORS (or somewhere else) and outputs a more accurate DGPS trace.
    • It is fine to cobble this together from existing tools provided:
      • The resulting software is well-integrated.
      • The component parts and associated licenses are documented clearly.
    • The best solution will access CORS (or something) as part of the software and not require the user to get the CORS information themselves.
    • Caveat:
      • This may not be possible using the output of the GPS receivers that we are using.
    • Deliverable
      • Software artifact and demo of it working, or
      • a paper clearly describing why you weren't able to get this working, what you could get to work and what options you investigated.
