  • Characteristics of State Tokens

    • Self-Describing. By inspecting the state token it is possible to determine:
      • what kind of state token it is (e.g., lock token)
      • what resource(s) it applies to
      • what state it represents
    • Client Generable. Clients should be able to generate a state token without having first communicated with a server (though they will need access to the state of the resource)
