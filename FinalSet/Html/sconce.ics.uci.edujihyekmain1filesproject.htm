Description

 

In a society increasingly concerned with the steady assault on electronic privacy, the need for privacy-preserving techniques is both natural and justified. This need extends to traditional security tools such as authentication and key distribution protocols.

 

 

 

 Secret Handshakes (Affiliation-hiding Authentication)

 

 

 

 

Secret handshakes allow group members' affiliations to be revealed only to members who hold specific roles in the group. For example, a CIA agent Alice might want to authenticate herself to Bob, but only if Bob is also a CIA agent. Moreover, if Bob is not a CIA agent, the protocol should not help Bob in determining whether Alice is a CIA agent or not.

The main characteristics of Secret Handshakes are:

Impersonation Resistance: prevent any party from impersonating as a group member unless the party is the  group member.

Detection Resistance: no information leaks from the protocol unless the observer is the same group member and participates the protocol. .

 

 

Group Signatures

 

Group signatures enable authorization of a user, without identifying the user, reducing the risk of information leaks and protecting the hoarding of surplus personal information. For example, these schemes may be applied to many systems in which user anonymity is required such as in anonymous credential systems, electronic voting systems, video/book rental systems, etc.

The main characteristics of Group Signatures are:

 Anonymity and Unlinkability: enable verification of individuals belonging to a group who have authorized access, without actually identifying the user by an ID or personal name.

 

 Taceability: The protocol allows for the identity of the signer to be discovered, in case of disputes, by a designated group authority that has some auxiliary information.
