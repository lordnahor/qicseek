  • Two types of timeout:

    • activity-based
      • a timer is set as soon as the lock is granted
      • every time a method is invoked on the resource, the timer is reset
      • when the timer runs out, the lock is removed
    • session-based
      • lock exists as long as the HTTP connection used to establish the lock is still connected
      • Example: Java program running in a browser
